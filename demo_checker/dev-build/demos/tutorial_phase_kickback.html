Building a quantum lock using phase kickback¬∂

Greetings, quantum adventurers! In this exciting tutorial, we‚Äôll be exploring the concept of quantum phase kickback, used in many quantum algorithms such as the Deutsch‚ÄìJozsa algorithm, and quantum phase estimation. Here, we‚Äôll be utilizing it to create a ‚Äúquantum lock‚Äù. Are you ready to dive into the quantum world and learn how to create an unbreakable lock? Let‚Äôs go!

Introduction to phase kickback¬∂

Phase kickback is a powerful quantum phenomenon that uses entanglement properties to allow for the transfer of phase information from a target register to a control qubit. It plays a vital role in the design of many quantum algorithms.

In a phase kickback circuit, an ancilla qubit is prepared in a superposition state using a Hadamard gate and it acts as a control qubit for a controlled unitary gate applied to the target register. When the target register is in an eigenstate of the unitary gate, the corresponding eigenvalue‚Äôs phase is ‚Äúkicked back‚Äù to the control qubit. A subsequent Hadamard gate on the ancilla qubit enables the extraction of the phase information through measurement.

If you want to know more about the details, do not hesitate to consult the node [P.1] of the Xanadu Quantum Codebook.

Setting up PennyLane¬∂

First, let‚Äôs import the necessary PennyLane libraries and create a device to run our quantum circuits. Here we will work with 5 qubits, we will use qubit [0] as the control ancilla qubit, and qubits [1,2,3,4] will be our target qubits where we will encode \(|\psi\rangle\).

import pennylane as qml
from pennylane import numpy as np

num_wires = 5
dev = qml.device("default.qubit", wires=num_wires, shots=1)

Building the quantum lock¬∂

Now let‚Äôs create the most formidable lock in the universe: the ‚Äúquantum lock‚Äù! Here our lock is represented by a unitary \(U\), which has all but one eigenvalue equal to 1. Our one ‚Äúkey‚Äù eigenstate has eigenvalue -1:

\[U|\text{key}\rangle = -|\text{key}\rangle\]

But how can we differentiate the ‚Äúkey‚Äù eigenstate from the other eigenstate when the information is contained in the phase? That‚Äôs where phase kickback comes in! When the correct eigenstate is input, the -1 phase imparted by \(U\) is kicked back to the ancilla, effectively changing its state from \(|+\rangle\) to \(|-\rangle\). Then the outcome of the measurement on the control qubit tells us whether the correct eigenstate was inputted or not. In this case, \(|1\rangle = H|-\rangle\) represents unlocking the lock, and \(|0\rangle = H|+\rangle\) represents failure. To make things simple, here we‚Äôll work with a lock-in computational basis. In this setting, the key corresponds to a binary encoded integer \(m\) , which will be our key eigenstate:

\[\begin{split}U|n\rangle = \begin{cases} -|n\rangle, & \text{if } n=m \\ |n\rangle, & \text{if } n\neq m \end{cases}\end{split}\]

We‚Äôll make use of FlipSign to build our lock:

def quantum_lock(secret_key):
    return qml.FlipSign(secret_key, wires=list(range(1, num_wires)))

Next, we need to prepare the corresponding eigenstate for a key we want to try out. Remember, the lock is only unlocked by the ‚Äúkey‚Äù eigenstate with eigenvalue -1. We‚Äôll make use of BasisState to build the key:

def build_key(key):
    return qml.BasisState(key, wires=list(range(1, num_wires)))

Now we‚Äôll put it all together to build our quantum locking mechanism:

@qml.qnode(dev)
def quantum_locking_mechanism(lock, key):
    build_key(key)
    qml.Hadamard(wires=0)  # Hadamard on ancilla qubit
    qml.ctrl(lock, control=0)  # Controlled unitary operation
    qml.Hadamard(wires=0)  # Hadamard again on ancilla qubit
    return qml.sample(wires=0)


def check_key(lock, key):
    if quantum_locking_mechanism(lock, key) == 1:
        print("Great job, you have uncovered the mysteries of the quantum universe!")
    else:
        print("Nice try, but that's not the right key!")

Opening the Quantum Lock¬∂

To open the quantum lock, we‚Äôll need the correct input state or ‚Äúquantum key‚Äù. Let‚Äôs see how the quantum system evolves when we input the right key.

We first apply a Hadamard to our control qubit:

\[\frac{|0\rangle|\text{key}\rangle + |1\rangle|\text{key}\rangle}{\sqrt{2}}\]

By applying the controlled unitary operation we get:

\[\frac{|0\rangle|\text{key}\rangle - |1\rangle|\text{key}\rangle}{\sqrt{2}} = |-\rangle|\text{key}\rangle\]

Finally, we apply a Hadamard to our control qubit again to get:

\[|1\rangle|\text{key}\rangle\]

And just like that, we‚Äôve uncovered the quantum secrets hidden by the lock. Let‚Äôs now crack open our quantum lock in code!

secret_key = np.array([0, 1, 1, 1])
lock = quantum_lock(secret_key)

check_key(lock, secret_key)

Great job, you have uncovered the mysteries of the quantum universe!

What happens with an incorrect quantum key?¬∂

Now, we‚Äôll try using the wrong key and see if we can still unlock the quantum lock. Will we be able to break through its quantum defenses? Let‚Äôs see how the quantum system evolves when we input the wrong key.

We first apply a Hadamard to our control qubit:

\[\frac{|0\rangle|\text{incorrect key}\rangle + |1\rangle|\text{incorrect key}\rangle}{\sqrt{2}}\]

Applying the controlled unitary operation, in this case, acts as the identity gate, hence we get:

\[\frac{|0\rangle|\text{incorrect key}\rangle + |1\rangle|\text{incorrect key}\rangle}{\sqrt{2}} = |+\rangle|\text{incorrect key}\rangle\]

Finally, we apply a Hadamard to our control qubit again to get:

\[|0\rangle|\text{incorrect key}\rangle\]

As you can see, we were unable to fool the almighty lock. Don‚Äôt believe me? See for yourself!

incorrect_key = np.array([1, 1, 1, 1])

check_key(lock, incorrect_key)

Nice try, but that's not the right key!
